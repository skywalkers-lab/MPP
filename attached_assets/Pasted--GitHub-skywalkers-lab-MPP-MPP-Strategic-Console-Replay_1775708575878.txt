현재 GitHub 저장소 skywalkers-lab/MPP를 기준으로, 기존 MPP Strategic Console과 Replay Console을 실제 운영 가능한 수준으로 고도화하라. 이번 작업은 단순한 UI 리스킨이 아니라, 가짜처럼 보이는 연출성 지표를 제거하고 실제 세션 데이터 기반 렌더링과 전략 입력 품질을 끌어올리는 구조 개선 작업이다. 기존 기능 중 유지할 것과 제거할 것을 명확히 구분하고, 모든 변경은 타입 안정성을 유지한 상태에서 구현하라. 작업이 완료되면 테스트와 빌드를 통과시키고, 최종 변경사항을 GitHub 원격 저장소에 푸시하라.

우선 1단계로, 라이브 전략 콘솔에서 가짜처럼 보이는 지표를 정리하라. “Strategy Delta vs P2”는 현재 실측 전략 델타가 아니라 연출용 수치처럼 보이므로 완전히 제거하라. 라이브 트랙맵은 현재 순위 기반 타원 배치 형태라 실제 서킷 데이터를 반영하지 못하므로, 실제 circuit path 기반 렌더링으로 교체하라. 트랙 레이아웃은 트랙별 path 데이터 또는 좌표 시퀀스를 사용해 SVG 혹은 canvas로 렌더링하고, 차량 위치 역시 실제 진행률이나 랩 거리 기반으로 표시하라. 타이어 열화 차트 또한 현재 단순 urgency 기반 시각효과에 가까우므로, 실제 모델 출력 기반 차트로 교체하라. 즉 최근 랩 데이터로 피팅된 열화 모델의 예측값, 기준선, 임계점, 추천 피트랩 등을 시각화하고, 더 이상 하드코딩된 가상 곡선을 사용하지 마라.

2단계로, Replay Console의 데모 성격을 제거하라. classification, event log, telemetry polyline, 서킷 관련 고정 샘플값, 게이지 샘플값 등 하드코딩된 데모 데이터를 전부 제거하라. Replay 화면은 반드시 실제 session snapshot과 stored replay frames를 기반으로 렌더링되어야 한다. 리플레이 타임라인 scrubber, 이벤트 마커, 분류표, 차량 위치, 속도/스로틀/브레이크/ERS 등의 수치는 저장된 프레임 데이터에 의해 변해야 하며, 정적인 예시 문자열이나 임의 polyline이 남아 있으면 안 된다. replay frame 저장 구조가 미비하다면, 세션 중 수집된 스냅샷을 시간축 기준으로 저장하고 재생 가능한 replay store를 먼저 설계한 뒤 그 위에 UI를 연결하라.

3단계로, 전략 입력 버퍼를 강화하라. 현재 전략엔진 입력에 들어가는 recentLapTimesMs는 지나치게 얕으므로, 최근 10랩에서 20랩 사이를 유지하는 ring buffer 구조를 도입하라. 이 버퍼에는 정상 레이스 랩만 들어가야 하며, 인랩, 아웃랩, 피트 통과 랩, Safety Car 구간, VSC 구간, 명백한 이상치 랩은 제외하라. 단순 last lap과 best lap만으로 열화 모델을 피팅하는 방식은 폐기하고, 최근 랩의 유효 샘플을 이용해 랩타임 열화 피팅을 고도화하라. 최소한 선형보다 나은 열화 추정이 가능하도록 다항식 또는 보다 안정적인 회귀 접근을 검토하되, 과적합 없이 설명 가능한 형태를 유지하라. 동시에 차량 손상도도 전략 입력과 UI 모두에서 명확히 볼 수 있게 하라. 프론트엔드에는 front wing, rear wing, floor, sidepod, engine, gearbox 등 손상 정보를 표시하고, 전략엔진에서도 손상으로 인한 페이스 저하나 피트 필요성을 반영할 수 있도록 입력 구조를 확장하라.

4단계로, 액션 버튼을 실제 기능과 연결하라. 현재 BOX THIS LAP, PUSH NOW, HARVEST MODE 등의 버튼은 장식에 가깝기 때문에, note/event/command bus와 연결하여 실제 동작 가능한 명령 체계로 바꿔라. 예를 들어 BOX THIS LAP은 전략 노트 추가나 command event 발행, PUSH NOW는 공격 모드 요청, HARVEST MODE는 에너지 세이브 요청처럼 명확한 command payload를 남기도록 하라. 버튼 클릭 시 UI 피드백만 바뀌는 것이 아니라, 백엔드 혹은 세션 이벤트 스트림에 기록이 남아야 한다. 이후 타임라인과 노트 패널, 이벤트 로그에서도 해당 액션이 추적 가능해야 한다.

추가로 라이브 텔레메트리 창의 비율과 레이아웃을 전면 점검하라. 현재 구성은 화면 크기에 따라 지표가 깨지거나 겹치거나 일부가 가려질 가능성이 있으므로, 반응형 레이아웃과 최소/최대 폭 제어를 재설계하여 어떤 주요 해상도에서도 핵심 지표가 가려지지 않도록 하라. 중앙 라이브 텔레메트리 영역, 좌측 상태 패널, 우측 전략 패널, 하단 이벤트 패널 사이의 비율을 조정하고, overflow 처리와 테이블 최소 너비도 함께 보정하라.

기능 정리 측면에서, “Virtual Garage” 기능과 “Overlay” 기능은 이번 개편 범위에서 제거하라. 관련 탭, 라우트, 버튼, 링크, placeholder, dead code, 설정 페이지 노출 항목까지 모두 정리하여 제품 구조를 더 단순하고 일관되게 만들라. 남길 필요가 없다면 UI에서 숨기는 수준이 아니라 실제 코드와 라우팅까지 정리하라.

구현 원칙은 다음과 같다. 첫째, 기존 타입스크립트 타입 안정성을 유지하고 any 사용을 피하라. 둘째, 실데이터 기반 렌더링과 연출용 임시 데이터는 명확히 구분하고, 이번 작업 후에는 라이브/리플레이 주요 화면에서 하드코딩된 시각용 샘플값이 남지 않게 하라. 셋째, UI만 바꾸지 말고 RelayServer, replay frame 저장 구조, strategy input pipeline, 프론트엔드 타입 계약까지 함께 정합성을 맞춰라. 넷째, 삭제 대상 기능은 미련 없이 정리하되, 현재 빌드와 테스트가 깨지지 않도록 안전하게 제거하라.

완료 기준은 다음과 같다. 라이브 콘솔에서 Strategy Delta vs P2가 제거되어 있을 것. 트랙맵이 실제 circuit path 기반으로 렌더링될 것. 타이어 열화 차트가 실제 모델 출력값을 표시할 것. Replay Console에서 classification, events, telemetry, gauges가 stored replay frames 기반으로 동작할 것. 최근 10~20랩 ring buffer가 도입되고 인랩/아웃랩/SC/VSC 제외 규칙이 반영될 것. 차량 손상도 정보가 UI와 전략 입력에 모두 연결될 것. 액션 버튼이 note/event/command bus에 실제 기록을 남길 것. Virtual Garage와 Overlay 관련 기능이 제거될 것. 라이브 텔레메트리 창 비율이 보정되어 주요 지표가 깨지거나 가려지지 않을 것. 마지막으로 전체 테스트와 빌드가 통과해야 하며, 변경사항을 적절한 커밋 메시지로 정리한 뒤 GitHub 원격 저장소에 push까지 수행하라.

작업을 시작하기 전에 현재 구조를 분석하고, 변경이 필요한 파일 목록과 작업 순서를 스스로 정한 뒤 바로 구현에 들어가라. 중간 결과는 단순 상태보고가 아니라 실제 수정 내용과 검증 결과 위주로 제시하라. 최종적으로는 수정된 파일 요약, 제거된 데모/가짜 지표 목록, 새로 연결된 데이터 흐름, 테스트 결과, 그리고 푸시한 커밋 해시를 함께 보고하라.
